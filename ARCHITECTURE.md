## 核心模块说明

### 1. 前端应用 (client/)

**技术栈**: React 18 + TypeScript + Vite + TailwindCSS

**主要功能**:
- 用户认证（登录/注册）
- 客服工作台（会话管理、消息收发）
- 实时通信（WebSocket）
- Web3 钱包连接
- 多语言切换
- 响应式设计（PC + 移动端）

**目录结构**:
```
client/src/
├── components/     # UI 组件
├── pages/         # 页面组件
│   ├── auth/      # 认证页面
│   ├── dashboard/ # 客服工作台
│   └── chat/      # 聊天界面
├── hooks/         # 自定义 Hooks
├── stores/        # Zustand 状态管理
├── services/      # API 服务
├── locales/       # i18n 语言包
└── utils/         # 工具函数
```

### 2. 后端应用 (server/)

**技术栈**: Node.js + Express + TypeScript + Prisma

**主要模块**:

#### 2.1 认证模块 (middleware/auth.ts)
- JWT Token 生成和验证
- 用户身份认证
- 角色权限控制

#### 2.2 Telegram 集成 (telegram/)
- **Bot API**: 使用 grammy 框架
  - Webhook 消息接收
  - 命令处理
  - 文件处理
  
- **Client API**: 使用 MTProto
  - 用户授权
  - 消息收发
  - 联系人同步
  
- **代理服务**: 解决国内访问限制
  - SOCKS5 代理
  - 连接池管理
  - 故障转移

#### 2.3 客服分发模块 (services/)
- 消息队列（Redis）
- 负载均衡算法
  - 轮询 (Round-Robin)
  - 最少连接 (Least Connections)
  - 随机分配 (Random)
- 客服状态管理
- 会话分配和转接

#### 2.4 AI 辅助模块 (ai/)
- 对接东京服务器 AI 服务
- 智能回复建议
- 消息翻译
- 情感分析
- 关键词提取

#### 2.5 WebSocket 模块
- Socket.IO 实现
- 实时消息推送
- 在线状态更新
- 打字指示器

### 3. 数据库设计 (prisma/schema.prisma)

**核心表**:

1. **users** - 系统用户（客服/管理员）
2. **agents** - 客服详细信息
3. **customers** - Telegram 客户信息
4. **conversations** - 会话记录
5. **messages** - 消息内容
6. **assignments** - 消息分派记录
7. **telegram_bots** - Bot 配置
8. **telegram_accounts** - Telegram 账号绑定
9. **wallets** - Web3 钱包
10. **language_preferences** - 语言偏好
11. **quick_replies** - 快捷回复模板
12. **system_logs** - 系统日志

### 4. 基础设施

#### 4.1 Docker 容器化
```yaml
services:
  - postgres:15-alpine    # 数据库
  - redis:7-alpine        # 缓存/队列
  - server (Node.js)      # 后端
  - client (Nginx)        # 前端
```

#### 4.2 CI/CD (GitHub Actions)
- 自动化测试
- 代码质量检查
- Docker 镜像构建
- 自动部署

## 数据流示例

### 场景 1: 用户发送消息到客服

```
1. Telegram 用户
   ↓
2. Telegram Bot (Webhook)
   ↓
3. 后端接收消息
   ↓
4. 保存到数据库 (PostgreSQL)
   ↓
5. 推送到消息队列 (Redis)
   ↓
6. 分发系统选择客服
   ↓
7. 通过 WebSocket 推送给客服
   ↓
8. 客服工作台显示消息
   ↓
9. 客服回复
   ↓
10. 保存到数据库
    ↓
11. 通过 Bot API 发送给用户
```

### 场景 2: AI 辅助回复

```
1. 收到用户消息
   ↓
2. 发送到 AI 服务 (东京服务器)
   ↓
3. AI 分析消息
   ↓
4. 返回建议回复
   ↓
5. 保存到消息记录
   ↓
6. 显示在客服工作台
   ↓
7. 客服可以选择采纳或修改
```

### 场景 3: Web3 钱包验证

```
1. 用户点击连接钱包
   ↓
2. RainbowKit 弹出钱包选择
   ↓
3. 用户选择钱包并签名
   ↓
4. 前端发送签名到后端
   ↓
5. 后端验证签名
   ↓
6. 绑定钱包地址到用户账户
   ↓
7. 返回验证成功
```

## 安全架构

### 1. 认证和授权
- JWT Token 认证
- Refresh Token 机制
- 基于角色的访问控制 (RBAC)

### 2. 数据安全
- 密码 bcrypt 加密
- Session 数据加密存储
- HTTPS/TLS 传输加密

### 3. API 安全
- CORS 配置
- Rate Limiting 速率限制
- Input Validation 输入验证
- SQL Injection 防护 (Prisma ORM)

### 4. WebSocket 安全
- Token 验证
- Origin 检查
- 消息频率限制

## 技术选型理由

| 技术 | 选择理由 |
|------|---------|
| React | 生态丰富，组件化，社区活跃 |
| TypeScript | 类型安全，提高代码质量 |
| Vite | 极速开发体验，现代构建工具 |
| Prisma | 类型安全 ORM，优秀 DX |
| PostgreSQL | 可靠，功能强大，ACID 兼容 |
| Redis | 高性能，多功能（缓存/队列） |
| Express | 轻量，灵活，中间件生态 |
| Socket.IO | 成熟 WebSocket 方案 |
| Zustand | 简单，轻量，无 boilerplate |
| TailwindCSS | 快速开发，一致性设计 |
| grammy | 现代 Bot 框架，TypeScript 友好 |
| wagmi/viem | 最佳 Web3 开发体验 |
